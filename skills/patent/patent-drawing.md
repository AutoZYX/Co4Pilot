# Patent Drawing Skill (专利附图绘制)

> 基于 Patent Agent（彪哥）与张玉新的迭代对话 + 10 篇已授权自动驾驶/AI 专利分析 + PT9 实际绘制反馈蒸馏而来。
> 版本：v2.1（2026-04-17 晚，基于 PT9 7 张图绘制反馈升级）
> 适用领域：自动驾驶、汽车、人工智能（PIL/Pillow 或 SVG 可覆盖的所有图形类型）

## v2.1 关键升级（基于 PT9 实战反馈）

张玉新在 PT9 绘制实测后给出 6 条硬性反馈，全部写入规范：

1. **图内不放图号/图名**（如"图 1 方法总流程图"）——这些由代理人在 Word 正文输入
2. **图四周零边距**（safety margin ≤6 px）——图要直接复制进 Word，不能被白边撑大
3. **不写底部注释**（如"注：三类指标互补…"）——说明性文字全部放正文，图下不写
4. **文本精确居中**（必须用 PIL `anchor="mm"`）——否则圆圈里的数字会跑到外面
5. **禁止重叠**（文字 vs 文字、文字 vs 图元素）——必要时用白底矩形遮盖背景虚线/曲线
6. **多列框严格均分**（`slot = total_w / n_cols`，每框宽 = slot - 间距）——避免列宽不一导致重叠

## 何时触发本 Skill

- 用户说"画专利附图""绘制发明专利配图""做一张专利说明书插图"
- 正在写技术交底书需要补配图建议
- 专利申请前需要补附图

## 核心原则（四条铁律）

1. **清楚 > 美观**——审查员 3 秒看懂优于视觉效果
2. **每图一意**——禁止一张图塞多个独立内容
3. **可单独使用**——不看文字说明，仅看图+图名能理解
4. **可验证输出**——每张图生成后必须像素级校验（见 §6）

## 一、八类附图决策树（PIL/Pillow 可覆盖）

| 类型 | 名称 | 方向 | 典型场景 |
|------|------|------|---------|
| A | 方法流程图 | 竖向 | S1→S2→S3 步骤序列 |
| B | 系统模块架构图 | 横/竖 | 分层/分模块的系统组成 |
| C | 结构/组成框图 | 横向 | 装置各部分关系 |
| D | 场景/应用示意图 | 横向整页 | 车辆+车道+传感器布局 |
| E | 数据流/信号流图 | 横向 | 传感器→处理器→执行器 |
| F | 风险/等级分布图 | 竖向 | 柱状对比、分布曲线+阈值 |
| G | 对比/结果表格 | 竖向 | 现有技术 vs 本发明 |
| H | 设备结构框图 | 横向 | 车载系统物理组成 |

### 关键决策规则：竖向 vs 横向

- **内容是"步骤序列"** → 竖向（A4 竖版，从上往下）
- **内容是"空间布局/系统架构/场景"** → 横向整页（A4 横版，从左往右）

### 不在 PIL 能力内（需转其他工具）

- 芯片内部晶体管级电路图
- CAD 级别机械零件工程图
- 复杂封装剖面图（多层填充）
- 精确时序图（需精确时间轴对齐）

## 二、页面与排版规范（v2.1）

### 2.1 尺寸与边距
- **宽度**：竖向 680–900 px（适配 A4 竖版）；横向整页 1200–1600 px（适配 A4 横版）
- **边距**：MARGIN = 6 px 安全留边（不做装饰性白边；用户要把图复制到 Word，白边 = 浪费版面）
- **零边距校验**：生成后检查画布四角/四边中点像素是否为白色且与内容距离 ≤ 6 px

### 2.2 图内禁写（v2.1 硬规则）
- ❌ **不画图号/图名**（如"图 1 方法总流程图"）——由代理人在 Word 正文写
- ❌ **不画底部注释**（如"注：三类指标互补…"）——说明性文字全部放正文
- ❌ **不画图例说明块**（除非图元素本身必须靠图例辨识，否则直接删）
- ❌ **不画数据汇总表**（如实施例的 S2/S3/S4/S5 数值汇总）——实施例文字在正文写

### 2.3 字体与颜色
- **字体**：正文 ≥16pt，标题 ≥18pt，**必须** Noto Sans CJK（否则中文显示豆腐块 □）
- **颜色**：纯黑白，不用灰度/阴影/渐变（符合 CNIPA+USPTO 双标准）
- **线条**：2pt 黑色实线，虚线仅用于阈值/辅助线

## 三、对齐规范 v2.1（⚠️ 最易错）

### 3.1 箭头公式（根因修复点）

```python
# ❌ 错误：累加间距 → 箭头穿过后续多个图框
arr(draw, cx, y_box_bottom, y_box_bottom + (i+1)*gap)

# ✅ 正确：从上一框底部画到下一框顶边之前（留 8 px 三角空间）
def arrow_v_down(draw, cx, y_start, y_end, head=8):
    draw.line([(cx, y_start), (cx, y_end - head)], fill=BLACK, width=2)
    draw.polygon([(cx, y_end),
                  (cx - head, y_end - head),
                  (cx + head, y_end - head)], fill=BLACK)
```

### 3.2 文本居中必须用 anchor="mm"（v2.1 新增）

```python
# ❌ 错误：基于 bbox 手动居中，偏移量导致小空间内字符跑出边界（圆圈/方块内）
w, h = text_size(draw, text, font)
draw.text((cx - w//2, cy - h//2), text, font=font)

# ✅ 正确：语义精确居中，不受 bbox 偏移影响
draw.text((cx, cy), text, fill=BLACK, font=font, anchor="mm")
```

PIL `anchor` 常用值：`"mm"` 中-中；`"lm"` 左-中；`"mt"` 中-上；`"mb"` 中-下。

### 3.3 圆圈编号必须用专用 helper

```python
def numbered_circle(draw, cx, cy, r, num, font):
    """圆形中放数字，anchor='mm' 精确居中。"""
    draw.ellipse([cx - r, cy - r, cx + r, cy + r], outline=BLACK, width=2)
    draw.text((cx, cy), str(num), fill=BLACK, font=font, anchor="mm")
```

### 3.4 多列布局严格均分

```python
# ✅ 按总宽均分，每框宽 = 列宽 - 间距（避免列宽不一导致重叠）
slot = total_w // n_cols
col_centers = [x1 + slot // 2 + i * slot for i in range(n_cols)]
box_w_each = slot - 16  # 保留 16 px 列间距
```

### 3.5 防遮挡（文字 vs 虚线/曲线）

```python
# 文字下方先画白底矩形遮住背景，再绘制文字
tw_, th_ = text_size(draw, text, font)
pad = 6
draw.rectangle([cx - tw_//2 - pad, cy - th_//2 - pad,
                cx + tw_//2 + pad, cy + th_//2 + pad], fill=WHITE)
draw.text((cx, cy), text, fill=BLACK, font=font, anchor="mm")
```

## 四、各类型图具体规范

### A. 竖向方法流程图（最常用）

- 每步骤一个矩形方框
- 标号和内容同一行：`S1. 数据预处理`
- 相邻方框间距 ≥25 px
- 箭头 2pt 粗，向下
- 分支逻辑用菱形判定框 + `是/否` 标注

### B. 系统模块架构图

- 三列或四列布局
- 顶部父框（系统名）向下分支到子模块
- 层级不超过 2 层（避免嵌套混乱）
- 数据流用箭头，信号类型标注在箭头旁（如"CAN""ETH""PCIe"）

### C. 结构/组成框图

- 横向排列
- 外层大框 + 内部子框表示组件包含关系
- 用引线+参考标号（如 301、302）指向各部件

### D. 场景/应用示意图（横向整页）

- 车道用两条水平线表示，车辆用圆角矩形
- 传感器视场用扇形/虚线框
- 标注距离（如"前车距 30 m"）
- 俯视图优先，除非必须展示高度关系

### E. 数据流/信号流图

- 横向从左到右
- 方框表示处理单元，箭头表示数据流向
- 信号类型标在箭头上方

### F. 风险/等级分布图

- **柱状图**（权重对比）：不用雷达图；PIL 绘柱状图更可靠
- **分布曲线**（危险分级）：简化折线 + 阈值虚线 + 误差棒 T 字形
- X 轴/Y 轴必须有标签 + 单位

### G. 对比/结果表格

- 表头加粗边框
- 列宽均匀
- 现有技术一列 + 本发明一列，勾 √ / 叉 ×

### H. 设备结构框图

- 横向
- 大矩形包子模块
- 标注各模块的输入/输出接口

## 五、PIL 代码骨架（可直接复用）

```python
from PIL import Image, ImageDraw, ImageFont

# 字体加载（必须 Noto Sans CJK，否则中文豆腐块）
FONT_PATH = "/System/Library/Fonts/PingFang.ttc"  # macOS
# FONT_PATH = "/usr/share/fonts/noto-cjk/NotoSansCJK-Regular.ttc"  # Linux
font_body = ImageFont.truetype(FONT_PATH, 16)
font_title = ImageFont.truetype(FONT_PATH, 18)

# 竖向流程图基础尺寸
W, H = 800, 1100
img = Image.new("RGB", (W, H), "white")
draw = ImageDraw.Draw(img)

def box(draw, x1, y1, x2, y2, text, font):
    draw.rectangle([x1, y1, x2, y2], outline="black", width=2)
    # 文本居中
    bbox = draw.textbbox((0, 0), text, font=font)
    tw, th = bbox[2]-bbox[0], bbox[3]-bbox[1]
    draw.text((x1 + (x2-x1-tw)//2, y1 + (y2-y1-th)//2),
              text, fill="black", font=font)

def arr(draw, cx, y_start, y_end, arrow_size=8):
    """箭头：从 y_start 画到 y_end，末端三角形向下"""
    draw.line([(cx, y_start), (cx, y_end)], fill="black", width=2)
    draw.polygon([(cx, y_end),
                  (cx - arrow_size, y_end - arrow_size),
                  (cx + arrow_size, y_end - arrow_size)],
                 fill="black")

# 步骤定义
steps = ["S1. 数据采集与预处理", "S2. 特征提取",
         "S3. 融合算法", "S4. 决策输出"]

box_h = 80
gap = 40
y = 30
for i, step in enumerate(steps):
    box(draw, 100, y, 700, y+box_h, step, font_body)
    if i < len(steps) - 1:
        # ✅ 正确箭头：从当前框底部画到 gap 中点
        arr(draw, 400, y + box_h + 2, y + box_h + gap - 5)
    y += box_h + gap

img.save("fig1_flowchart.png")
```

## 六、校验规范（发送前必须执行）

### 6.1 箭头完整性校验

```python
# 对每个箭头间隙，在线段中点检测像素是否为黑（0）
for (y_bottom, y_end) in arrow_gaps:
    mid = (y_bottom + y_end) // 2
    px = img.getpixel((center_x, mid))
    assert px == (0, 0, 0) or px == 0, f"Arrow missing at y={mid}"
```

### 6.2 内容边界校验

- 扫描图像四边，确认没有内容被切掉
- 扫描四周 5px 外围，应全为白色

### 6.3 无多余元素校验

- 箭头不应穿过任何图框
- 没有孤立线条（非连接箭头的短线段）
- 没有重复边（箭头与图框边重叠）

### 6.4 视觉抽查

打开 PNG 后人眼 3 秒判断：
- [ ] 不看文字能否大致理解结构？
- [ ] 中文字是否正常（不是 □）？
- [ ] 打印 A4 后字是否可读？

## 七、常见错误清单 v2.1（14 条）

| # | 错误 | 后果 | 修复 |
|---|------|------|------|
| 1 | 箭头终点用累加间距 `(i+1)*gap` | 箭头穿过多个框 | 改为单 `gap` |
| 2 | 场景/架构图用竖向 | A4 打印字过小 | 空间布局→横向整页 |
| 3 | 没装 Noto Sans CJK | 中文显示 □ 豆腐块 | 预装字体+指定路径 |
| 4 | 边缘留白太多 | 占用 Word 版面 | MARGIN ≤ 6 px |
| 5 | 图里有图号/图名 | Word 里重复标题 | ❌ 图内不写，正文写 |
| 6 | 图下有"注："等注释 | 和说明书正文冗余 | ❌ 正文承担 |
| 7 | 嵌套子框 ≥3 层 | 布局混乱 | 最多 2 层 |
| 8 | 不校验直接发送 | 发现错为时已晚 | 必须像素校验 |
| 9 | 雷达图 | PIL 绘制不可靠 | 改柱状图 |
| 10 | 圆圈数字 bbox 居中偏移 | 数字跑出圆圈 | 用 `anchor="mm"` |
| 11 | 多列 box 宽度靠手算 | 列重叠 | `slot = total_w / n_cols` 均分 |
| 12 | 文字压到虚线/曲线 | 阅读困难 | 先画白底矩形再写字 |
| 13 | 实施例图带大块数据汇总 | 图与正文重复 | 只画视觉内容，数值放正文 |
| 14 | 车辆/图元标签贴图元素太近 | 互相遮挡 | 离最近元素 ≥ 8 px |

## 八、用户指令格式（提高命中率）

触发本 Skill 时，用户越详细越好。最佳指令格式：

```
【专利附图】
图名：[一句话图标题，对应说明书附图说明]
类型：[A/B/C/D/E/F/G/H 或描述]
方向：[竖向 / 横向整页]
核心元素：
 - [元素1 + 标号（如有）]
 - [元素2 + 标号]
 - ...
数据流/连接关系：
 - [A → B，数据类型：xxx]
 - [B → C，信号：CAN]
特殊要求：
 - [如："需要 5 个核心模块"、"前车距 30 m"、"含阈值虚线"]
参考编号方案：
 - [如："主模块 100，子模块 101-105"]
```

### 最小可行指令（也能接）

```
画图8：俯视场景图。包括本车（前向摄像头+激光雷达）、前车、左右车道线、前方 30 m 障碍物。横向整页。
```

## 九、与其他 Skill 的关系

- `patent-drafting.md`：负责文字（说明书、权利要求、附图说明），本 Skill 负责图片本身
- `patent-mining.md`：提供创新点，本 Skill 据此决定画几张图、画哪些
